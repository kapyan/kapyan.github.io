    mysql状态管理

    命令 作用
    show processlist; 查看数据库正在执行的sql语句，可能无法看全完整的SQL
    show full processlist; 查看正在执行的完整SQL语句，完整显示
    set global key_buffer_size = 1024*1024*16; 不重启数据库调整数据库参数，立即生效，重启失效
    show variables; 查看数据库配置参数信息
    show variables like ‘%log_bin%’; 模糊匹配查看配置信息
    kill ID 杀死SQL线程
    show session status; 查看当前会话的数据状态信息
    show global status; 查看整个数据库运行的状态信息
    show engine innodb status; 显示innodb引擎的性能状态

    数据库管理SQL

    新建数据库

    create database 数据库名 character set 字符集 collate 排序规则;

    创建用户

    create user ‘用户名‘@’主机域’ identitied by ‘密码’;

    用户授权

    grant 权限 on 库名.表名 to ‘用户名‘@’主机域’ identified by ‘密码’ [with grant option];

    all表示所有权限，多个权限之间用‘,’隔开，授权时用户不存在会自动创建，加‘with grant option’表示该用户可以将自己拥有的权限授权给其他用户。

    权限列表：

    权限 说明
    SELECT 查询（数据）
    INSERT 插入（数据）
    UPDATE 修改（数据）
    DELETE 删除（数据）
    CREATE 创建（数据库、表等对象）
    DROP 删除（数据库、表等对象）
    RELOAD 重载
    SHUTDOWN 关闭
    PROCESS 进程
    FILE 文件
    REFERENCES 参考资料
    INDEX 索引
    ALTER 修改（数据库、表对象）
    SHOW DATABASES 查看数据库
    SUPER 超级权限
    CREATE TEMPORARY TABLES 创建临时表
    LOCK TABLES 锁表
    EXECUTE 执行
    REPLICATION SLAVE 从复制权限
    REPLICATION CLIENT 从客户端复制
    CREATE VIEW 创建视图
    SHOW VIEW 查看视图
    CREATE ROUTINE 创建存储过程
    ALTER ROUTINE 修改存储过程
    CREATE USER 创建用户
    EVENT 事件
    TRIGGER 触发器
    CREATE TABLESPACE 创建表空间

    插入数据

    insert into 表名 (字段名1,字段名2,…) values (值1,值2,…),(值1,值2,…)…

    更新数据

    update 表名 set 字段名=值 where 条件表达式

    删除数据

    delete from 表名 where 条件表达式

    清空表

    truncate table 表名

    truncate 与不带where的delete效果相同，但是速度比delete高。
